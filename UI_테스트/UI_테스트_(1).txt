I. UI 테스트 기법
1. 휴리스틱 평가 (Heuristic Evaluation)
- 1. 전문가에 의해 이론과 경험 (Heuristic) 을 근거, 일련의 규칙 정의, 평가 대상의 규칙 준수 여부 확인하는 평가 방법
- 2. 사용성 문제 찾아내기 위한 사용성 공학 방법
- 3. 목적 -> 휴리스틱 가이드라인, 대상의 문제점을 발견하고 디자인에 반영
- 4. 결과물 -> 평가보고서

2. 페이퍼 프로토타입 (Paper Prototype) 평가
- 1. 제품 전반적인 컨셉, 더 자유롭게 의견 공유하여 발전시킬 수 있는 평가 방법
- 2. 실제 출시될 제품의 디자인을 미리 경험해 봄
- 3. 결과물 -> 프로토타입

** UI 테스트 수행 순서
- 1. 사용성 테스트 계획 (UI 테스트 기법 선정)
- 2. 테스트 환경 구축 (선정된 기법에 적합한)
- 3. 사용성 테스트 계획서 작성
- 4. 사용성 테스트 수행
- 5. 사용성 평가 분석서 작성, 이슈 도출
- 6. 테스트 결과 보고 (개선방안 및 수정계획 수립)

3. 선호도 (Preference) 평가
- 1. 중요도에 따른 선호도를 예측하기 위해 사용되는 평가 방법
- 2. 사용자 Needs (요구) 파악
- 3. 사용자 감성 파악
- 4. 결과물 -> 선호도 평가 (점수, 순위, 태도기반 선호도, 속성기반 선호도)

4. 성능 (Performance) 평가
- 1. Task (작업, 기능) 별로 평가, 성능 개선하는 평가 방법
- 2. 학습성, 효율성, 기억용이성, 오류, 만족도 평가
- 3. 종류
  = 1. 학습성 -> 쉽게 학습할 수 있는가
  = 2. 효율성 -> 신속하게 사용 가능 여부
  = 3. 기억용이성 -> 사용한 기능 능숙하게 사용 가능 여부
  = 4. 오류 -> 오류 적고, 사용자가 상황을 쉽게 극복할 수 있는가
  = 5. 만족도 평가 -> 즐겁고 만족
  

II. 사용성 (Usability)
1. 사용자가 업무를 쉽고 편리하게 수행
2. 특정한 맥락의 사용에서의 효과성, 효율성, 만족도
3. UPA -> 사용자의 피드백을 받는 제품 개발 형식
4. Jakob Nielsen -> 학습용이성, 사용효율성, 기억용이성, 최소한의 에러, 주관적 기쁨

** 유용성 (Usefulness)
- 1. 사용자의 업무 수행, 얼마나 정확하게 수행할 수 있는지 나타낸 것
- 2. ex) 그래서 무엇을 제공할 것인가를 의미


III. 사용성 테스트 (Usability test)
1. 정의 및 특징
- 1. UI 테스트에 대한 보편적인 방법
- 2. 구현된 UI 직접 사용, 문제점 도출
- 3. 실제 배포 전 최종적으로 검증
- 4. Example) 웹 사이트 UI 사용성 테스트
  = 1. 사이트 구성 메뉴 문제점, 요구사항 반영 여부 점검
  = 2. 선호할 만한 웹 사이트 기능 테스트
  = 3. 사용자 새 요구사항 반영되도록 하는 것

2. 종류
- 1. 파일럿 테스트 -> 컴퓨터 프로그램, 소규모로 시험 작동 해보는 방법 => 모의 시행
- 2. 심층 인터뷰 -> 1명의 응답자, 1:1 면접 => 조사원 면접 및 분석 능력에 따라 조사 결과 신빙성, 타당성 크게 달라짐
- 3. 포커스 그룹 인터뷰 -> 6 ~ 12명 정도 소비자 선발, 관련 토론 진행
- 4. 맥락적 인터뷰 -> 특정 상황, 맥락, 특정 행동을 관찰 및 조사

3. 효과
- 1. 사용자 제품 만족도 향상
- 2. 노력 대비 높은 효율성 및 개발 생산성 대비 낮은 에러

4. 사용성 테스트 주의 사항
- 1. 감성적인 측면
- 2. 기본적인 기능 테스트 (기본 기능 구현 정확 여부 검사) 누락 발생하지 않도록 주의
- 3. 테스트 인원 자율적으로 테스트 가능 여부
- 4. 자세한 기능 설명, 구현 방법
- 5. 테스트가 오랜 시간 진행될 경우, 지연 사유 체크

** 옵저베이션 (Observation)
- 1. 어떻게 구성, 어떻게 해서 생기는가의 사실을 있는 그대로 확인
- 2. 실험 대상이나 과정에 인위적인 간섭을 가하지 않는 것

** 테스트 신뢰성
- 1. 측정치, 측정 기준 신뢰할 수 있을 정도로 정확성, 안정성, 일관성 보유 여부 확인
- 2. 테스트 신뢰성에 문제가 있는 테스트 결과는 아무런 의미 X, 활용 X


IV. 사용자 중심 디자인 (User Centered Design)
1. 정의 및 특징
- 1. 사용자를 중심, 철학적 접근 방법이자 디자인 사상
- 2. 디자인은 인간 사용자의 편의와 복지의 실현에 초점
- 3. 사용자가 어떻게 사용하고 있고 어떻게 사용하게 될지 분석, 예상을 하는 Design Research 를 활용

2. 인터렉션 디자인 (Interaction Design)
- 1. 인터렉션 -> Digital System 과 사람이 주고 받는 일련의 의사소통 과정
- 2. 인터렉션 디자인 -> 사람의 행동과 이에 반응하는 시스템의 절차 설계
- 3. HCI 의 인터렉션 -> 사용자에게 최적의 경험 제공하는 기본 단위
- 4. 시스템 디자인에서 인터렉션 -> 시스템의 행동적인 측면에 초점

3. HCI (Human Computer Interaction or Interface)
- 1. 인간과 Computer 간 상호 작용에 대한 연구
- 2. System Design 및 평가하는 과정을 다룬 학문

4. 감성 공학 (Human Sensibility Ergonomics)
- 1. 인체의 특징과 감성을 제품 설계에 최대한 반영시키는 기술
- 2. 인간 중심의 설계
- 3. HCI 설계에 인간 특성과 감성 반영


V. 사용성 평가 분석서
- Test 자료 수집, 사용성 평가 분석서를 작성하고 이를 토대로 이슈 사항 도출 가능

VI. Task 성공 Matrics
1. 정의
- 1. Task (기능, 작업) 의 성공, 실패 여부 분석
- 2. 사용성 Test 에서 가장 일반적이고 보편적인 방법

2. 에러 Matrics
- 1. 사용자가 범한 실수를 수집하여 분석
- 2. Task 진행 중 얼마나 많은 실수 있었는지, 제품 어떤 부분에서 발생했는지, 디자인에 따라 어떤 종류의 Error 발생 빈도, 일반적으로 무엇이 어떻게 유용한지 확인

3. 효율성 Matrics
- 1. Task 를 완료하는데 필요한 단계나 행동 개수에 의해 측정
- 2. Task 마다 걸린 평균 시간, Task 성공률 비율에 따라 측정
- 3. Task 완료 위해선 걸리는 시간 뿐 아니라, 노력, 물리적 노력의 양을 중요하게 여길 때 효과적

4. 시간 기반 Task Matrics
- 1. Task 완료 시간 (Task 시간) 을 수집하여 분석
- 2. 스톱워치로 기록, 자동화 툴 사용, 시계 끄고 켜기 방법

5. NEM 기법
- 1. 초심자 (처음 써보는 사람), 숙련자 (설계, 개발에 참여한 사람) 를 사용성 Test 에 참여
- 2. Test 수행 시간 비교하여 문제점 파악

** Usability Matrics
- 1. 특정 현상, 사물을 측정하고 평가하는 기법
- 2. 개인 경험, 신뢰성 있는 방식 요구
- 3. 사용자 경험 데이터를 수집 및 분석 가능

** 데이터 이력 기록기 (Data Logger)
- 아날로그 입력을 디지털 수로 변환, 자동적으로 기록


VII. UI 관련 용어
1. UI Design (User Interface Design)
- 1. 사람과 접하는 면을 설계하고 디자인
- 2. IA (Information Architecture) 설계, Interaction 설계, GUI Design

** Interface
- 1. System 입출력 장치, 사물 간 또는 가물과 인간 간 의사소통 가능, 물리적, 가상적인 매개체
- 2. IA (Information Architecture, 정보 구조, Site Map)
  = 사이트 구성 처리 내용, 메뉴 구조 표현 -> 사이트 구조 파악할 수 있도록 하는 것

- 3. GUI (Graphical UI) -> Graphic / 마우스 기반 Interface

2. UX Concept Review (Concept : 차별화 전략)
- UX 전략, UI Conception, GUI Conception 결과물 -> Review 하여 개선

** UI Conception, GUI Conception
- 1. 주요 화면 (Key Screen) 에 대한 Concept 설계 및 가시화
- 2. UI 설계, 업무형태 유사, UX Concept 가시화, 실제 설계 전 검증하는데 목적 있음
- 3. Concept Board 제작 (Concept 검토 가능)

3. Concept Model
- 1. 추상적인 Concept 들 사이의 관계 보여주는 Diagram
- 2. 다양한 아이디어들을 간편하게 시각화
- 3. 생각의 과정 효율적으로 이끌어 줌
- 4. 문서 작성자, 문서 활용하는 사람들에게 복잡한 Idea 들과 관계들을 고민해 볼 수 있는 기회가 됨

4. Mental Model
- 1. 사람들의 행동 동기, 사고 과정 뿐만 아니라, 그들이 행동하는 감성적, 철학적 배경에 대해서도 깊이 이해 가능
- 2. 에스노그래피 자료를 의미상 가까운 것끼리 모아 놓은 친화도

** 에스노그래피 (Ethnography)
- 사람들 (Ethnos) + 기록 (Grapho) -> 다른 문화 관찰, 기록하는 연구 방법에서 시작됨

5. Card Sorting
- Idea 와 Concept 을 작은 Card 에 작성, 분류 및 정렬하여 정보를 구조화

6. 정량적 리서치 (Quantitative Research) 
- 1. 통제된 조건에서 수치화 된 데이터
- 2. 경쟁사 사이트, 벤치마킹 사이트 또는 이전 디자인과 비교
- 3. 설문 형태, 한 회당 20명 정도 참가자 수행
- 4. 예) 동선, 무엇, 몇 분

7. 정성적 리서치 (Qualitative Research) 
- 1. 행동 원인, 의도, 상황 파악 (수치화 X)
- 2. 사용자 행동과 관련된 정황, 인사이트를 얻기 위한 방법
- 3. 반응, 개선
- 4. 한 회 당 5 ~ 8명 구성, 한 회 이상 수행하는 것이 이상적
