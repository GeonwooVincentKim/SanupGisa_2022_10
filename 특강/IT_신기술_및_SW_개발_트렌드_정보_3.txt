I. 멤리스터 (Memristor)
- 1. Memory + Register 합성어, 기존의 상태를 모두 기억하는 소자, Register, 커패시터, 인덕터, 전자 회로 구성 요소, 차세대 기억 소자, 회로 등에 응용
- 2. Memristor 를 Computer System Memory 등에 이용, Energy 소모와 부팅 시간을 획기적으로 줄일 수 있음

1. RAID (Redundant Array of Inexpansive Disk , 복수 배열 독립 Disk)
- 1. Computer 저장장치, 하나의 대형장치 대신 다수의 일반 Hard 를 Array 로 구성
- 2. Data 분할해서 분산 저장, 다중화 -> Disk 고장 대비하여 Data 의 안정성 높인 Computer 저장 장치
- 3. 값싼 Disk 를 여러 개 묶어 대용량 저장공간을 만들고자 함

2. 4K 해상도 (4K Resolution) - 약 4,000 Pixel, X축 해상도, Digital Image -> 극장용 Digital 영화 등의 표준
3. Phablet (패블릿) - Phone + Tablet 합성어, 5인치 이상 대화면 SmartPhone (ex -> Galaxy Note)

4. C형 USB (USB Type-C, USB-C, Universal Serial Bus Type-C)
- 단자 위아래 동일한 24핀의 USB, USB-IF (USB Implementers Forum) 에 의해 2014년 USB Type-C 규격 (1.0) 발표

II. MEMS (멤스, Micro-Electro-Mechanical Systems, 초소형 정밀기계 등)
- 21세기 최대 유망 기술, 초미세 기계 구조물을 만드는 기술 (손톱 크기의 H/D), Micro Meter (100만 분의 1미터) 이하 정밀도

1. 트러스트존 (TrustZone)
- 1. 독립적인 보안 구역 따로 두고, 중요 정보를 보호, H/W 기반의 보안 기술
- 2. 암(ARM: Advanced RISC Machine) 사에 개발
- 3. 일반 구역 (Normal World), 보안 구역 (Secure World)

2. M-DISC (엠디스크, Millennial DISC)
- 1. 한 번의 기록만으로 자료 영구 보관할 수 있는 광 저장 장치
- 2. Digital Video Disk (DVD), Blue-Ray Disk (DVD 보다 빠르고 용량 큼)

III. Big Data (빅 데이터)
- 1. 처리하기 어려운 막대한 양의 정형, 비정형 Data 집합
- 2. 정부, 포털 등이 Big Data 를 효과적으로 분석 -> 미래 예측해 최적 대응방안 찾고, 수익 연결, 새로운 가치 생성
- 3. 용량(Volumn), 다양성 (Variety), 속도 (Velocity)
- 4. 서울지역 카드결제 정보 이용하여 야간 버스노선 개설

1. 하둡 (Hadoop)
- 1. 대량의 자료 (Big Data) 를 처리할 수 있는 큰 Computer Cluster (많은 양의 계산 진행, Data 저장하기 위해 여러 대 Computer 를 하나로 묶어 놓은 것)
- 2. 분산 응용 Program 을 지원하는 자유 Java S/W Framework (Open Source 기반의 분산 Computing Platform)
- 3. 구글 MapReduce 개발 (2004년), 여러 대 Computer 를 마치 하나인 것처럼 묶어 대용량 Data 처리하는 기술 (비용 절감 가능)

2. 맵리듀스 (MapReduce)
- 1. 대용량 Data 를 분산 처리하기 위한 목적, 개발된 Programming Model
- 2. Google 에 의해 고안, 대용량 Data 처리를 위한 병렬 처리 기법

3. Tajo (타조)
- 1. Hadoop 기반의 대용량 Data 웨어하우스, 2010년 고려대학교 Prototype 에서 시작
- 2. MapReduce 사용 X, 구조화 질의 언어 (SQL) 를 사용, Hadoop 분산 File System (HDFS: Hadoop Distributed File System) File 을 바로 읽어내는 기술

4. 데이터웨어하우스 (Data Warehouse)
- 1. 정보 (Data) + 창고 (Warehouse) 합성어, 기업 정보 자산을 효율적으로 활용
- 2. Data 시계열적 (시간적 순서에 따른 배열) 축적, 통합 목표로 하는 기술의 구조적, 통합적 환경
- 3. 방대한 조직 내에서 분산 운영되는 각각의 DB 관리 System 들 효율적 통합, 조정 및 관리
- 4. 효율적인 의사 결정 System 을 위한 기초 제공

5. Broad Data
- 1. Big Data -> 기대 자료, Broad Data -> 기업 Marketing 에 보다 효율적인 다양한 정보
- 2. 기존 Data 에 새로운 가치를 더하는 Data (SNS 활동, 위치 정보)

IV. Digital Archiving (디지털 아카이빙)
- 1. Digital 정보 자원을 장기적으로 보존하기 위한 작업, Meta Data 만들어 DB 화하는 작업
- 2. 늘어나는 정보 자원의 효율적인 관리, 이용을 위해 필요한 작업
- 3. Archiving -> File 보관

1. Data Diet (데이터 다이어트)
- 1. Data 삭제 X, 압축하고, 겹친 정보는 중복 배제, 새로운 기준에 따라 나누어 저장하는 작업
- 2. 중복 데이터 압축, 제거해주는 S/W 를 사용하면 저장량을 5분의 1로 줄일 수 있음

2. LOD (Linked Open Data, 개방형 링크드 데이터)
- 1. Web 상 모든 Data, DB 공개 및 연결
- 2. 인터넷 식별자 (URI) 를 통해 Data 를 서로 연결 (연계 Data)함, Web 에 공개, 연계, 공유
- 3. 연계 Data (Linked Data), Open Data 결합됨

V. Digital 포렌식 (Digital Forensic)
- Computer 법의학, 휴대폰, PDA, PC, Server 등 Data 수집 분석하는 Digital 수사 과정 -> 포렌식센터가 계설되어 있음

1. 공통 평가 기준 (CC, Common Criterial) - 정보 보호 제품 (ex -> Anti-Virus) 의 평가 기준을 규정한 국제 표준 (ISO 15408)
2. RTO (Recovery Time Objective, 목표 복구 시간) - 비상사태, 업무중단 시점부터 업무가 복구되어 다시 정상가동 될 때까지의 시간

3. RPO (Recovery Point Objective, 목표 복구 시점)
- 복구할 수 있는 기준점, 복구가 필요한 업무에 대해 어느 시점까지 Data 필요한가에 따라 시점 정함

4. 트러스트존 (TrustZone)
- 1. 독립적인 보안 구역 따로 두고, 중요 정보를 보호, H/W 기반의 보안 기술
- 2. 암(ARM: Advanced RISC Machine) 사에 개발
- 3. 일반 구역 (Normal World), 보안 구역 (Secure World)

VI. ZigBee (지그비)
- 1. IEEE 802.15 표준 기반, 소형, 저전력 Digital Radio -> 개인 통신망 구성, 통신하기 위한 표준 기술
- 2. 양방향 무선 개인 영역 통신망 (WPAN) 기반 Home Network 및 무선 센서망에서 사용되는 기술
- 3. 낮은 Data 를, 적은 배터리 소모 Network 안전성

** IEEE 802 주요 표준 규격
- 1. 802.1 -> 전체 구성, OSI 참조 모델과의 관계, 통신망 관리
- 2. 802.2 -> 논리 Link 제어 (LLC) 계층
- 3. 802.3 -> CSMA / CD   -4. 802.4 -> Token Bus   - 5. 802.5 -> Token Ring
- 6. 802.6 -> 도시형 통신망 (MAN), DQDB (이중 Bus 통신망)
- 7. 802.7 -> 광대역 LAN  - 8. 802.11 -> 무선 LAN (WiFi)
- 9. 802.15 -> WPAN / Bluetooth / ZigBee

1. VoIP (Voice over Internet Protocol, 인터넷 전화)
- 1. IP Network 를 통해 음성 Digital Packet 형태로 전달
- 2. Traffic 많아지면 통화 품질 떨어질 수 있음

2. NAT (Network Address Translation, Network 주소 변환)
- 1. 사내 개별 주소 (사설 IP), 정식 IP 주소(공인 IP) 를 상호 변환
- 2. 사설 Network 에 속한 여러 개 Host 가 하나의 공인 IP 주소 사용
- 3. IP 주소 재기록하면서 Router 를 통해 Network Traffic 을 주고 받는 기술

3. 워터마킹 (Watermarking, Digital Watermarking)
- 지적재산 저작권 보호를 위해 저작권자 또는 판매자 정보를 Content 에 숨겨 넣는 기술

4. Finger Printing
- WaterMarking 과 동일, Contents 를 구매하는 사용자 정보 삽입, Content 불법 배포자를 추적하는데 사용

5. Wm-Bus (Wireless Meter-Bus, 무선 미터 버스)
- 1. 전기, 가스, 수도 등 원격 검침 위한 Smart Metering, AMI (Advanced Metering)
- 2. 서브-기가헤르츠 대역 (sub-GHz), OverHead 가 매우 적고 Battery 수명을 최장 20년까지 지속

** Smart Grid -> Smart + 전기나 가스 공급하는 배급망 or 전력망(Grid) 합쳐짐
** Smart Metering -> Smart Grid System 선행, 자동 사용량 측정 System
** AMI (Advanced Metering Infrastructure, 첨단 Energy 검침 Infra System)
- Energy 사용 정보 측정, 수집, 저장, 분석, 이를 활용하기 위한 총체적인 System

VII. PIMS (Personal Information Management System, 개인정보 관리 System)
- 각종 정보를 효율적으로 관리해주는 총합 System, 인증 부여 제도, PIMS 인증제 도입

VIII. PRM (Partner Relationship Management, 파트너 관계 관리)
- 파트너들과 협업하는 Solution, 파트너를 통해 간접적으로 고객 관리

IX. WIPI (Wireless Internet Platform for Interoperability, 위피)
- 한국형 무선 Internet Platform (OS) 표준 규격

X. WAP (Wireless Application Protocol, 무선 응용 통신 규약)
- 1. 무선망, Internet 연동하기 위해 Protocol, 공개된 표준
- 2. HyperText 전송 규약 (HTTP) 을 지원하지 않기 때문에 WAP Gateway 를 사용해야 하므로, 비용이 많이 듬 (단점)

XI. 데이터 지역화 (Data Locally)
- 1. 저장하는 방법, 필요한 위치에 기꺼이 Data 를 배치하는 것
- 2. 시간적 지연성 -> 가장 최근에 참조된 Memory 가 다시 참조 될 가능성이 높음 (for, while)
- 3. 공간적 지연성 -> 가장 최근에 참조된 Memory 와 가까운 Memory 가 다시 참조 될 가능성이 높음 (ex, A[0], A[1])

XII. GPIB (General-Purpose Interface Bus, 범용 Interface Bus)
- 1. Computer 를 포함한 전자장비의 입출력을 제어, 범용의 병렬 Interface Bus (IEEE 488)
- 2. Computer 와 측정 장비를 연결, 정보 교환 및 제어

XIII. 키오스크 (Kiosk)
- 1. 고객의 편의, Computer 자동화 System
- 2. 발권, 구매, 등록을 대행하는 단말기

XIX. OTT (Over-The-Top, 오버더톱 Service)
- 개방된 Internet, Media Content 를 제공하는 Service (Netflix, Wavve)

1. Media Big Bang
- 1. 정보 통신의 발달로 새로운 Media 가 등장, Media 환경 변화를 행성 대폭발을 의미하는 Big Bang 에 비유한 표현
- 2. ex) Computer 와 방송이 결합한 Smart TV, Netflix

2. MQTT (Message Queuing Telemetry Transport, 엠큐티티)
- 1. Push 기술 기반의 경량 메시지 전송 Protocol
- 2. 송신자 -> 특정 메시지 발행, 수신자 -> 메시지 구독, 발행-구독 방식

3. GPN (Global Production Network, 글로벌 생산 Network)
- 생산 공정이 여러 나라에 분산된 생산, 유통 및 소비, 운영 및 거래

4. Scoop
- 관계형 DB, 하둡 (BigData, 분산 처리) 사이, Data 이관 지원하는 Tool

5. FIN (Fused Indoor localizatioN, 융합 실내 측위 기술)
- 1. Mobile Network 신호 활용, 사용자 위치를 파악하는 기술
- 2. Navigation 위치파악

6. DSA (Digital Signature Algorithm, 전자 서명 알고리즘)
- 미국 국립 표준 기술 연구소 (NIST), 전자 서명 표준안으로 개발된 엘가말 암호 방식 기반

** 전자 서명 (Digital Signature)
- 1. Data 출처와 무결성을 확인, 암호화 알고리즘으로 계산되어 Data 에 첨부되는 Code
- 2. Data 생성원, 무결성을 검증 가능, 제 3자에 의한 Data 위조 등을 보호가능

** 엘가말 암호 방식 (Elgamal Encryption Scheme)
- 공개 Key 암호 방식의 하나, 최초의 공개 Key 암호, 암호문 길이가 평문 길이 2배로 되는 결점있음

7. Apnic (에이피닉, Asia Pacific Network Information Center) - 아시아, 태평양 Network 정보 센터
8. VLAN (Virtual Local Area Network, 가상 랜) - 논리적 LAN 구성할 수 있는 기술
9. 포스퀘어 (Foursquare)
- 위치 기반 Social Networking Service (SNS)

** SNS 누리 소통망 Service -> 사람들이 연결, 온라인 기반의 개인 간 Social Network 만들고 운영하는데 초점을 맞춘 Service (FaceBook, Instagram 등)

10. Metaverse - Universe + Meta 합성어, 진보된 개념
11. N2OS (Neutrailzed Network Operating System)
- 1. 한국전자통신연구원(ETR) 개발, Network 정비에 탑재되어 H/W 자원 관리하는 S/W
- 2. 최적 장비 전달, 개방형 구조 S/W Framework, Open Platform 을 지향

12. SSO (Single Sign-On, 싱글 사인 온)
- 1. 하나의 System 에서 인증에 성공, 다른 System 에 대한 접근 권한
- 2. 한 번의 로그인 만, 각종 System, Internet Service 에 접속하게 해주는 보안 응용 Solution

13. PKI (Public Key Infrastructure, 공개 Key 기반 구조)
- 1. 공개 Key 암호 System, 정보 보호 표준 방식
- 2. ITU-T 의 X.509 방식, 비X.509 방식의

14. SOS (Security Operating Service, 보안 운영 Service)
- 1. 종합적 보안 관리를 원격으로 제공하는 Service
- 2. 보안 Solution 관리, 보안 Network 장비 관리, Server 관리, 침해 예방 및 대응, 백업/복구, 보안 Log 분석, 보안 Solution 임대, 관제 등 Service
